# UX Review — Gym Management System (2026-04-01)

## Overall Score: 7.5/10

### Strengths
- Kiosk design genuinely suited to tablet + wet hands context
- Foundations students surfaced everywhere (amber badges, blocked promotions, briefings)
- WordPress admin conventions respected throughout
- Quick Check-In sticky bar always reachable
- Location awareness persistent and pervasive
- Belt progression visualization effective

### CRITICAL Issues
1. **Kiosk search input has no visible label** — kids and older members won't know what to type. Add "Type your name to check in" as visible text.
2. **Belt progression dots use color as sole differentiator** — WCAG 1.4.1 violation. Add text labels or distinct shapes for color-blind users (~8% of males).
3. **CSS typo: `border-radius:50` missing `%`** — `UserProfileRank.php:230`. Filled stripe dots render as squares.
4. **No AJAX timeout in kiosk JavaScript** — if API is slow, kiosk spinner blocks all check-ins indefinitely. Add 10-second timeout with recovery message.

### HIGH Issues
5. **Quick Check-In typeahead has no "no results" feedback** — coach sees nothing on misspelled name
6. **Location toggle uses full page reload** — 2-5 second disruption mid-class. Convert to AJAX.
7. **Admin typeahead lacks ARIA combobox attributes** — WCAG 4.1.2 violation
8. **Individual Promote button has no confirmation** — accidental promotions are irreversible
9. **Member dashboard inline CSS not cached** — move to external file
10. **No Foundations badge on kiosk check-in results** — Foundations students could select inappropriate classes

### MEDIUM Issues
11. Plan name shows price instead of product name in some cases
12. "0 days ago" instead of "Today" for same-day promotions
13. At-risk member list has no action links
14. Kiosk lacks client-side double-tap prevention
15. History tab filter pattern inconsistent with Promotions page
16. Location selector doesn't explain what switching does
17. "Rec." column header is cryptic

### LOW Enhancements
18. Show streak/badges on kiosk success screen
19. Add keyboard shortcut for Quick Check-In focus
20. Add date range selector to Trends tab
21. Warmer empty states on member dashboard
22. Reduce jQuery dependency in location selector

### Accessibility Summary
- WCAG 1.3.1: Missing visible labels (kiosk)
- WCAG 1.4.1: Color-only belt differentiation
- WCAG 1.4.3: Some contrast failures (empty state text on white)
- WCAG 2.4.7: Focus indicators may be invisible on dark admin bar
- WCAG 4.1.2: Admin typeahead missing ARIA combobox pattern
- Positive: screen-reader labels, aria-pressed on toggles, prefers-reduced-motion
